# Product Requirements Document (PRD)

> This file will be generated during `/project-setup` based on your answers.

## Expected Sections

- **Problem Statement**: What problem does this solve? Why does it matter?
- **Target Users**: Who is this for? What are their needs and pain points?
- **Core Features (MVP)**: The minimum set of features for a first working version.
- **User Stories**: Key user flows described as "As a [user], I want [action] so that [outcome]."
- **Success Criteria**: How do you know the project is working? Measurable outcomes.
- **Out of Scope**: What this project explicitly does NOT do in v1.
- **Monetization** (optional): Revenue model if applicable.
- **Open Questions**: Anything still unresolved after the scoping conversation.
