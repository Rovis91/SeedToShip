# Technology Stack

> This file will be generated during `/project-setup` after the PRD is defined. The tech stack is derived from the project requirements, not chosen in advance.

## Expected Sections

- **Frontend**: Framework, styling, state management (if applicable).
- **Backend**: Language, framework, API style (if applicable).
- **Database**: Type, provider, ORM (if applicable).
- **Infrastructure**: Hosting, CI/CD, deployment strategy.
- **Third-Party Services**: Auth, payments, email, analytics, etc.
- **Dev Tools**: Linter, formatter, testing framework, package manager.
- **Rationale**: Why this stack was chosen over alternatives.
- **Alternatives Considered**: Other options and why they were rejected.
