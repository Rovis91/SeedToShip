# Coding Constraints & Conventions

> This file will be generated during `/project-setup` after the tech stack is defined. Constraints are stack-specific and derived from the chosen technologies.

## Expected Sections

- **File & Folder Structure**: How the project should be organized.
- **Naming Conventions**: Files, variables, functions, components, routes.
- **Architecture Patterns**: e.g., server components by default, repository pattern, etc.
- **Error Handling**: How errors should be caught, logged, and surfaced.
- **Testing Requirements**: What must be tested, minimum coverage, testing patterns.
- **Code Style**: Formatting rules, import ordering, max file length.
- **Security Rules**: Input validation, auth patterns, secrets management.
- **Performance Guidelines**: Lazy loading, caching, bundle size limits.
- **Git Conventions**: Commit message format, branch naming, PR requirements.
- **Documentation Rules**: When and how to document code.
