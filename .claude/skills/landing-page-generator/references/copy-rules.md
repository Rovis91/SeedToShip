# Copy Rules

Sole owner of: banned words, body copy execution, tone-by-audience, length budgets, FAQ writing patterns. Hero formula and CTA labels live in SKILL.md (Step 4) so they load by default.

The page should read like it was written by someone with skin in the game, not a marketing department. Short sentences. Real verbs. Specifics over adjectives.

---

## Length budgets (verify at critique pass)

| Page type | Total meaningful words |
|---|---|
| Waitlist / pre-launch | 80–150 |
| Lead magnet / ebook | 150–250 |
| Standard SaaS / service | 300–500 |
| Enterprise / complex product | 500–700 |

If over budget, the page is explaining instead of selling. Cut.

## Banned words

Page contains no instance of these unless followed by specifics that earn the word.

| Banned word/phrase | Why | Replace with |
|---|---|---|
| Empower | empty, AI-slop signal | the actual outcome |
| Unlock | empty | the actual outcome |
| Leverage | corporate, vague | "use" |
| Seamlessly | always a lie | concrete integration description, or omit |
| Cutting-edge / state-of-the-art | unprovable | the specific tech, or omit |
| Best-in-class / world-class | self-praise without proof | a metric or testimonial |
| Revolutionize / transform | overused | the specific change |
| All-in-one / one-stop-shop | confession of unfocus | name 2–3 specific things it does |
| Solutions (alone) | filler word | what it actually is |
| Powerful | meaningless without specifics | the specific capability |
| Easy / simple | unprovable claim | "Setup in 5 minutes," "No code required" |
| Game-changer / next-gen | empty hype | the actual change |

This is a hard scan at the critique pass.

## Body copy execution

- **Second person.** "You" beats "users," "teams," "businesses."
- **Active voice, real verbs.** "We send" beats "messages are sent."
- **Cut every adjective that isn't doing real work.** "Fast" without a number is empty. "3× faster than [competitor]" is a claim.
- **One idea per sentence.** Max 3 lines per paragraph.
- **Specifics over generics.** "Save 4 hours/week" beats "Save time." If you don't have the number, name what changes concretely.
- **Customer language, not internal jargon.** Use the words customers say in calls, reviews, support tickets.

---

## Persuasion principles

1. **Specificity** — every claim has a number, a name, or a concrete example. Vague is uncredible.
2. **Loss-frame for pain, gain-frame for solution** — describe the cost of the status quo before the upside.
3. **Concrete proof beats clever copy** — one named testimonial with a number > three lines of clever positioning.
4. **Risk reversal** — every paid CTA needs an exit (free trial, money-back, cancel anytime, no credit card).
5. **Single conversion goal** — one CTA repeated. Two competing CTAs = visitor decides not to decide.
6. **Anchor first** — for pricing, lead with the recommended plan; the eye anchors there.

## Tone calibration by audience

| Audience | Tone signals |
|---|---|
| Developers | Technical specificity, code examples, no fluff, dry humor OK, GitHub links over testimonials |
| Enterprise B2B | Authoritative, ROI-led, named decision-makers in proof, security/compliance visible |
| SMB / solo founders | Direct, scrappy, $-savings + time-savings, founder-to-founder voice |
| Designers / creatives | Visual leads, less copy, opinionated aesthetic |
| Consumer | Emotional outcome, lifestyle imagery, reviews, prominent pricing |
| Technical buyers in non-tech orgs | Bridge language — translate technical capabilities into business outcomes |

## FAQ writing pattern

- **Q**: phrase the objection in the visitor's voice — direct, specific, real.
- **A**: 1–3 sentences. Lead with the answer, follow with proof or specifics, end with risk reversal if relevant.

**Avoid:**
- Vague reassurance ("We take your data seriously")
- Legalese
- Made-up softball questions ("What makes you different?")
- Walls of text

**Use:**
- "Even if..." framing for edge cases
- Specifics (numbers, integration names, timeframes)
- Risk reversal where it lowers the stakes

For category-by-category objection dismantling, see `objection-library.md`.
