# Copy Rules

The page should read like it was written by someone with skin in the game, not a marketing department. Short sentences. Real verbs. Specifics over adjectives.

## Length budgets (verify before handoff)

| Page type | Total page words |
|---|---|
| Waitlist | 60–120 |
| Lead magnet | 120–200 |
| Standard SaaS | 250–400 |
| Enterprise | 400–600 |

If over budget, the page is explaining instead of selling. Cut.

## Banned words

No instance of these unless followed by specifics that earn the word.

| Banned | Replace with |
|---|---|
| Empower | the actual outcome |
| Unlock | the actual outcome |
| Leverage | "use" |
| Seamlessly | concrete integration description, or omit |
| Cutting-edge / state-of-the-art | the specific tech, or omit |
| Best-in-class / world-class | a metric or testimonial |
| Revolutionize / transform | the specific change |
| All-in-one / one-stop-shop | name 2–3 specific things |
| Solutions (alone) | what it actually is |
| Powerful | the specific capability |
| Easy / simple | "Setup in 5 minutes," "No code required" |
| Game-changer / next-gen | the actual change |

Hard scan at pre-handoff verification.

## Body copy execution

- Second person ("you") — never "users," "teams," "businesses."
- Active voice. "We send" beats "messages are sent."
- Cut every adjective that isn't specific. "Fast" alone is empty. "3× faster than [competitor]" is a claim.
- One idea per sentence. Max 3 lines per paragraph.
- Specifics > generics. "Save 4 hours/week" beats "Save time."
- Customer language, not internal jargon.

## Persuasion principles

1. **Specificity** — every claim has a number, a name, or a concrete example.
2. **Loss-frame for pain, gain-frame for solution** — cost of status quo before the upside.
3. **Concrete proof beats clever copy** — one named testimonial with a metric > three lines of positioning.
4. **Risk reversal** — every paid CTA needs an exit (trial, money-back, cancel anytime, no credit card).
5. **Single conversion goal** — one CTA repeated.
6. **Anchor first** — for pricing, lead with the recommended plan.

## Tone calibration

| Audience | Tone signals |
|---|---|
| Developers | Technical specificity, code examples, no fluff, dry humor OK, GitHub links over testimonials |
| Enterprise B2B | Authoritative, ROI-led, named decision-makers in proof, security/compliance visible |
| SMB / solo founders | Direct, scrappy, time-and-money savings, founder-to-founder voice |
| Designers / creatives | Visual leads, less copy, opinionated aesthetic |
| Consumer | Emotional outcome, lifestyle imagery, reviews, prominent pricing |
| Technical buyers in non-tech orgs | Bridge language — translate technical capabilities into business outcomes |

## FAQ writing pattern

- **Q**: phrase the objection in the visitor's voice — direct, specific, real.
- **A**: 1–3 sentences. Lead with the answer, follow with proof or specifics, end with risk reversal if relevant.

**Avoid:** vague reassurance, legalese, made-up softball questions, walls of text.

**Use:** "Even if..." framing, specifics (numbers, integration names, timeframes), risk reversal.
